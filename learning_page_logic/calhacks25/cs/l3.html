String Fundamentals

Strings are sequences of case-sensitive characters. They're one of Python's fundamental data types and support a variety of operations.

String Length with len()

The len() function returns the number of characters in a string, including letters, digits, spaces, and special characters.

String Length Example Python 
s = "abc"
len(s)  # evaluates to 3
✨ Key Insight

The len() function is your first introduction to functions in Python. Functions are reusable procedures that perform specific tasks. We'll explore how to write your own functions in the next lecture!

Indexing & Slicing

String Indexing

Since strings are sequences, we can access individual characters using indexing. Python uses zero-based indexing, meaning the first character is at position 0.

Indexing Examples Python 
s = "abc"

# Positive indexing (from start)
s[0]   # → "a" (first character)
s[1]   # → "b" (second character)
s[2]   # → "c" (third character)

# Negative indexing (from end)
s[-1]  # → "c" (last character)
s[-2]  # → "b" (second to last)
s[-3]  # → "a" (third to last)
⚠️ Index Out of Bounds

Attempting to access an index beyond the string's length will raise an IndexError. For example, s[20] on a 3-character string will cause an error.

String Slicing

Slicing allows you to extract substrings using the notation [start:stop:step]. This powerful feature lets you manipulate strings in sophisticated ways.

Slicing Examples Python 
s = "abcdefgh"

# Basic slicing [start:stop]
s[3:6]      # → "def" (indices 3, 4, 5)

# Slicing with step [start:stop:step]
s[3:6:2]    # → "df" (every 2nd character)

# Full string copy
s[::]        # → "abcdefgh" (same as s[0:len(s):1])

# Reverse string
s[::-1]      # → "hgfedcba" (step backwards)

# Backwards slicing
s[4:1:-2]   # → "ec" (from index 4 to 2, step -2)
💡 Pro Tip

If you're unsure what a slice does, try it in your Python console! Experimentation is a great way to build intuition. The notation [::-1] is a common Python idiom for reversing sequences.

String Immutability

Strings in Python are immutable — once created, they cannot be modified. This is a crucial concept that applies to several Python data types.

Immutability Example Python 
s = "hello"

# This will raise an error!
s[0] = 'y'  # ❌ TypeError: 'str' object does not support item assignment

# Instead, create a new string
s = 'y' + s[1:len(s)]  # ✅ s is now "yello"
🔒 Understanding Immutability

When you "modify" a string, Python actually creates a new string object in memory and rebinds the variable to it. The original string remains unchanged (until garbage collected). This concept becomes important when working with more complex data structures.

Loops with Strings

For loops aren't limited to numbers — they can iterate over any sequence, including strings. This makes string processing elegant and readable.

Two Approaches to String Iteration

Approach 1: Index-Based Iteration Python 
s = "abcdefgh"

for index in range(len(s)):
    if s[index] == 'i' or s[index] == 'u':
        print("There is an i or u")
Approach 2: Direct Character Iteration (Pythonic!) Python 
s = "abcdefgh"

for char in s:
    if char == 'i' or char == 'u':
        print("There is an i or u")
🐍 Pythonic Code

The second approach is more "Pythonic" — it's cleaner, more readable, and follows Python's philosophy of simplicity. When you can iterate directly over elements rather than indices, do so!

Example: Robot Cheerleaders

Let's look at a fun example that demonstrates string iteration in action:

Robot Cheerleader Program Python 
an_letters = "aefhilmnorsxAEFHILMNORSX"

word = input("I will cheer for you! Enter a word: ")
times = int(input("Enthusiasm level (1-10): "))

# Iterate directly over characters (Pythonic!)
for char in word:
    if char in an_letters:
        print("Give me an " + char + "! " + char)
    else:
        print("Give me a " + char + "! " + char)

print("What does that spell?")
for i in range(times):
    print(word, "!!!")
🤖 Program Output

If you enter "robots" with enthusiasm level 6, the program will cheer each letter ("Give me an R! R", "Give me an O! O", etc.) and then shout "robots !!!" six times. It's a playful demonstration of string iteration and the in operator for membership testing.

Guess-and-Check Algorithms

Now we transition from string manipulation to algorithmic problem-solving. The guess-and-check method (also called exhaustive enumeration) is one of the simplest algorithmic approaches.

The Concept

Guess-and-Check Requirements

  • You can guess a value for the solution
  • You can check if the solution is correct
  • You systematically try values until you find the answer or exhaust all possibilities

Example: Finding Cube Roots

Let's apply guess-and-check to find the cube root of a number. We'll start with a simple version and progressively improve it.

Simple Cube Root (Perfect Cubes Only) Python 
cube = 8

for guess in range(cube + 1):
    if guess**3 == cube:
        print("Cube root of", cube, "is", guess)
⚠️ Limitation

This simple version has a problem: if the cube isn't perfect (like 9 or 10), the program silently fails without any output. Not very user-friendly!

Improved Version

Let's enhance our algorithm to handle negative numbers and provide feedback for non-perfect cubes:

Enhanced Cube Root with Error Handling Python 
cube = 27

for guess in range(abs(cube) + 1):
    # Stop once we've passed the answer
    if guess**3 >= abs(cube):
        break

# Check if we found a perfect cube
if guess**3 != abs(cube):
    print(cube, 'is not a perfect cube')
else:
    # Handle negative cubes
    if cube < 0:
        guess = -guess
    print('Cube root of ' + str(cube) + ' is ' + str(guess))
✨ Key Improvements
  • Efficiency: We use break to stop searching once we've passed the answer
  • Negative numbers: Using abs() allows us to handle negative cubes
  • User feedback: We inform the user when a cube isn't perfect

Approximate Solutions

What if we don't need an exact answer? What if "close enough" is good enough? This is where approximate solutions shine.

The Approach

Instead of looking for exact matches, we:

  • Start with an initial guess (often 0)
  • Increment by a small value (e.g., 0.0001)
  • Stop when we're "close enough" to the answer
📊 The Epsilon Parameter

We define "close enough" using epsilon (ε), a small threshold value. If |guess³ - cube| < ε, we've found an acceptable solution.

Approximate Cube Root Algorithm Python 
cube = 27
epsilon = 0.01
guess = 0.0
increment = 0.0001
num_guesses = 0

# Keep guessing until close enough
while abs(guess**3 - cube) >= epsilon and guess <= cube:
    guess += increment
    num_guesses += 1

print('num_guesses =', num_guesses)

if abs(guess**3 - cube) >= epsilon:
    print('Failed on cube root of', cube, "with these parameters.")
else:
    print(guess, 'is close to the cube root of', cube)

Trade-offs

⚖️ Accuracy vs. Speed
  • Smaller increment → More accurate, but slower
  • Larger epsilon → Faster, but less accurate
  • The boundary condition (guess <= cube) prevents infinite loops when we overshoot
Why We Need the Boundary Check

Imagine searching for the cube root of 10,000 with increment 0.01 and epsilon 0.1. You might step from "too small" to "too large" without ever landing in the epsilon range. The guess <= cube condition prevents infinite loops in such cases.

Bisection Search

Now for the most elegant algorithm of the three: bisection search. This method dramatically reduces the number of guesses needed by systematically halving the search space.

The Guessing Game

Imagine guessing a number between 0 and 100. Instead of guessing 1, 2, 3, 4... you could:

  1. Guess 50 (middle of 0-100)
  2. If too low, guess 75 (middle of 50-100)
  3. If too high, guess 62 (middle of 50-75)
  4. Continue halving the interval...
🎯 The Power of Bisection

With this method, you can guess any number from 0 to 100 in at most 7 guesses (log₂(100) ≈ 6.64). Compare that to potentially 100 guesses with linear search!

The Algorithm

Bisection Search for Cube Root Python 
cube = 27
epsilon = 0.01
num_guesses = 0
low = 0
high = cube
guess = (high + low) / 2.0

while abs(guess**3 - cube) >= epsilon:
    if guess**3 < cube:
        # Guess too low, search upper half
        low = guess
    else:
        # Guess too high, search lower half
        high = guess
    
    # New guess is midpoint of remaining interval
    guess = (high + low) / 2.0
    num_guesses += 1

print('num_guesses =', num_guesses)
print(guess, 'is close to the cube root of', cube)

Convergence Analysis

How Fast Does Bisection Converge?

  • First guess: N/2 (half the search space)
  • Second guess: N/4 (quarter of original space)
  • k-th guess: N/2ᵏ
  • Convergence: O(log₂N) steps
📈 Logarithmic Growth

Bisection search converges in logarithmic time. This means doubling the search space only adds one more guess! For a search space of 1,000,000, you'd need only about 20 guesses.

Important Considerations

⚠️ Limitations
  • Monotonicity required: Bisection only works when the function varies monotonically (consistently increases or decreases)
  • Numbers less than 1: The code shown works for cubes ≥ 1. For 0 < x < 1, the cube root is actually greater than x, so you'd need to adjust the search bounds
  • Negative numbers: Requires special handling (use absolute values and adjust sign at the end)
🎓 Challenge Problems

Try modifying the bisection code to:

  • Handle negative cubes correctly
  • Work with numbers between 0 and 1 (hint: set high = 1 when cube < 1)

Algorithm Comparison

Algorithm Time Complexity Accuracy Best Use Case
Guess-and-Check O(N) Exact (for integers) Small search spaces, integer solutions
Approximate Solution O(N/increment) Adjustable (epsilon) When "close enough" is acceptable
Bisection Search O(log N) Adjustable (epsilon) Large search spaces, monotonic functions

🎓 Key Takeaways

String Manipulation

  • Strings are immutable sequences of characters
  • Use len() to get string length
  • Indexing starts at 0; negative indices count from the end
  • Slicing syntax: [start:stop:step]
  • Iterate directly over characters for more Pythonic code

Algorithmic Approaches

  • Guess-and-Check: Systematically try all possibilities (O(N))
  • Approximate Solutions: Find "good enough" answers using epsilon thresholds
  • Bisection Search: Halve the search space each iteration for O(log N) performance

Problem-Solving Wisdom

  • Choose your algorithm based on the problem constraints
  • Consider trade-offs between accuracy and speed
  • Bisection search is powerful but requires monotonic functions
  • Always handle edge cases (negative numbers, values < 1, etc.)
