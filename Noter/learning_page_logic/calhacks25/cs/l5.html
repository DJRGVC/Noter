Introducing Tuples

Immutable Ordered Indexed

Tuples are compound data types - they're collections of other data types. Think of strings as sequences of characters; tuples are sequences of anything.

Key Characteristics

  • Ordered sequence of elements
  • Can contain mixed types (integers, floats, strings, etc.)
  • Immutable - once created, cannot be modified
  • Represented with parentheses ()
Creating and Using Tuples Python 
# Empty tuple
te = ()

# Tuple with mixed types
t = (2, "mit", 3)

# Indexing (starts at 0)
t[0]  # evaluates to 2

# Concatenation
(2, "mit", 3) + (5, 6)  # evaluates to (2, "mit", 3, 5, 6)

# Slicing
t[1:2]  # evaluates to ("mit",) - note the comma!
t[1:3]  # evaluates to ("mit", 3)

# Length
len(t)  # evaluates to 3

# Attempting to modify (ERROR!)
t[1] = 4  # TypeError: tuples are immutable

⚠️ The Singleton Tuple Gotcha

A tuple with one element requires a trailing comma: ("mit",)

Without the comma, ("mit") is just a string in parentheses, not a tuple!

Practical Uses of Tuples

1. Swapping Variables Elegantly

Remember the old way of swapping variables? Tuples make it beautifully simple:

❌ The Old Way
temp = x
x = y
y = temp
✅ The Tuple Way
(x, y) = (y, x)

2. Returning Multiple Values from Functions

Functions can only return one object, but that object can be a tuple containing multiple values!

quotient_and_remainder.py Python 
def quotient_and_remainder(x, y):
    """Returns both quotient and remainder as a tuple"""
    q = x // y  # Integer division
    r = x % y   # Remainder
    return (q, r)

# Unpack the returned tuple
(quot, rem) = quotient_and_remainder(4, 5)
print(quot)  # 0
print(rem)   # 4

How It Works

4 // 5 = 0
+
4 % 5 = 4
→
(0, 4)

3. Iterating Over Tuples

A powerful real-world example: extracting data from structured tuples.

get_data.py - Analyzing Tuple Data Python 
def get_data(aTuple):
    """
    aTuple: tuple of tuples (int, string)
    Returns: (min_int, max_int, unique_strings_count)
    """
    nums = ()     # Empty tuple for numbers
    words = ()    # Empty tuple for unique words
    
    for t in aTuple:
        # Add the integer (first element)
        nums = nums + (t[0],)
        
        # Add unique strings only
        if t[1] not in words:
            words = words + (t[1],)
    
    min_n = min(nums)
    max_n = max(nums)
    unique_words = len(words)
    
    return (min_n, max_n, unique_words)

# Real-world example: Taylor Swift song data! 🎵
tswift = ((2014, "Katy"),
          (2014, "Harry"),
          (2012, "Jake"), 
          (2010, "Taylor"), 
          (2008, "Joe"))

(min_year, max_year, num_people) = get_data(tswift)
print(f"From {min_year} to {max_year}, Taylor Swift wrote songs about {num_people} people!")
# Output: From 2008 to 2014, Taylor Swift wrote songs about 5 people!

Introducing Lists

Mutable Ordered Indexed

Lists are similar to tuples, but with one crucial difference: they're mutable. You can change, add, and remove elements after creation.

Lists vs Tuples

Feature Tuples Lists
Syntax (1, 2, 3) [1, 2, 3]
Mutability ❌ Immutable ✅ Mutable
Use Case Fixed collections Dynamic collections
Performance Faster Slightly slower
Creating and Indexing Lists Python 
# Empty list
a_list = []

# List with mixed types
L = [2, 'a', 4, [1, 2]]

# Length
len(L)  # 4

# Indexing
L[0]      # 2
L[2] + 1  # 5
L[3]      # [1, 2] - a list within a list!
L[4]      # IndexError: list index out of range

# Using variables as indices
i = 2
L[i-1]    # 'a' (same as L[1])

The Power of Mutability

Unlike tuples, you can modify list elements directly:

Modifying Lists Python 
L = [2, 1, 3]
L[1] = 5  # ✅ This works!
# L is now [2, 5, 3]

Memory Visualization

[2, 1, 3]
→
[2, 5, 3]

Same object, modified in place

List Operations

Iterating Over Lists

Two approaches: iterate over indices or iterate over elements directly.

Method 1: Using Indices
total = 0
for i in range(len(L)):
    total += L[i]
Method 2: Direct Iteration ⭐
total = 0
for element in L:
    total += element

✨ More Pythonic!

Adding Elements

Adding to Lists Python 
L = [2, 1, 3]

# Append: add to end (mutates list)
L.append(5)
# L is now [2, 1, 3, 5]

# Concatenation: create new list
L1 = [2, 1, 3]
L2 = [4, 5, 6]
L3 = L1 + L2
# L3 is [2, 1, 3, 4, 5, 6]
# L1 and L2 unchanged

# Extend: add multiple elements (mutates list)
L1.extend([0, 6])
# L1 is now [2, 1, 3, 0, 6]

🔍 What's the Dot?

The dot notation (L.append()) means we're calling a method on the list object. Methods are functions that belong to specific object types. We'll explore this more in future lectures on Object-Oriented Programming!

Removing Elements

Removing from Lists Python 
L = [2, 1, 3, 6, 3, 7, 0]

# Remove by value (first occurrence only)
L.remove(2)
# L is now [1, 3, 6, 3, 7, 0]

L.remove(3)
# L is now [1, 6, 3, 7, 0] - only first 3 removed

# Delete by index
del(L[1])
# L is now [1, 3, 7, 0]

# Pop: remove and return last element
last = L.pop()
# last is 0, L is now [1, 3, 7]

⚠️ Important: Return Values

Most list mutation methods return None, not the modified list!

Exception: pop() returns the removed element.

Converting Between Lists and Strings

String ↔ List Conversions Python 
s = "I<3 cs"

# String to list (each character)
list(s)
# ['I', '<', '3', ' ', 'c', 's']

# Split on a character
s.split('<')
# ['I', '3 cs']

# List to string
L = ['a', 'b', 'c']
''.join(L)
# 'abc'

'_'.join(L)
# 'a_b_c'

Sorting and Reversing

Sorting Lists Python 
L = [9, 6, 0, 3]

# sorted(): returns new sorted list (doesn't mutate)
new_list = sorted(L)
# new_list is [0, 3, 6, 9]
# L is still [9, 6, 0, 3]

# .sort(): mutates the list (returns None)
L.sort()
# L is now [0, 3, 6, 9]

# .reverse(): mutates the list
L.reverse()
# L is now [9, 6, 3, 0]

Mutability, Aliasing, and Side Effects

🚨 Critical Concept Alert!

This is one of the most important and tricky concepts in Python. Use Python Tutor to visualize what's happening!

The Celebrity Analogy

Let's understand aliasing with a real-world example:

Justin Bieber: One Person, Many Names

Consider a person (object) with attributes:

singer • rich

This person has multiple aliases:

  • ✦ Justin Bieber
  • ✦ The Bieb
  • ✦ JBeebs

If we add an attribute to any alias:

singer • rich • troublemaker

ALL aliases now refer to the person with the new attribute!

Aliasing in Lists

Aliasing Example Python 
# Create a list
warm = ['red', 'yellow', 'orange']

# Create an ALIAS (not a copy!)
hot = warm

# Modify through one alias
hot.append('pink')

# BOTH variables show the change!
print(hot)   # ['red', 'yellow', 'orange', 'pink']
print(warm)  # ['red', 'yellow', 'orange', 'pink']

Memory Diagram

warm ──┐
└──→ ['red', 'yellow', 'orange', 'pink']
hot ───┘

Both variables point to the SAME list object!

Cloning Lists

To avoid aliasing issues, create a clone (copy) of the list:

Cloning a List Python 
cool = ['blue', 'green', 'grey']

# Clone using slice notation
chill = cool[:]

# Modify the clone
chill.append('black')

# Original is unchanged!
print(chill)  # ['blue', 'green', 'grey', 'black']
print(cool)   # ['blue', 'green', 'grey']

Memory Diagram: Cloning

cool ──→ ['blue', 'green', 'grey']
chill ──→ ['blue', 'green', 'grey', 'black']

Two separate list objects!

Sorting: Mutation vs. New List

.sort() - Mutates
warm = ['red', 'yellow', 'orange']
result = warm.sort()

print(warm)    # ['orange', 'red', 'yellow']
print(result)  # None
sorted() - Returns New
cool = ['grey', 'green', 'blue']
result = sorted(cool)

print(cool)    # ['grey', 'green', 'blue']
print(result)  # ['blue', 'green', 'grey']

Side Effects and Nested Lists

Nested Lists

Lists can contain other lists, and aliasing still applies!

Nested Lists and Side Effects Python 
warm = ['yellow', 'orange']
hot = ['red']

# Create a list containing another list
brightcolors = [warm]
brightcolors.append(hot)

print(brightcolors)
# [['yellow', 'orange'], ['red']]

# Modify the original hot list
hot.append('pink')

# brightcolors is affected!
print(brightcolors)
# [['yellow', 'orange'], ['red', 'pink']]

⚠️ Danger: Mutating While Iterating

Common Pitfall

Modifying a list while iterating over it can lead to unexpected behavior!

The Problem Python 
def remove_dups(L1, L2):
    for e in L1:
        if e in L2:
            L1.remove(e)  # ❌ Modifying while iterating!

L1 = [1, 2, 3, 4]
L2 = [1, 2, 5, 6]
remove_dups(L1, L2)

print(L1)  # [2, 3, 4] - Expected [3, 4], but got [2, 3, 4]!

✅ The Solution: Clone First!

The Fix Python 
def remove_dups_fixed(L1, L2):
    L1_copy = L1[:]  # ✅ Clone the list first!
    for e in L1_copy:
        if e in L2:
            L1.remove(e)  # Modify original, iterate over copy

L1 = [1, 2, 3, 4]
L2 = [1, 2, 5, 6]
remove_dups_fixed(L1, L2)

print(L1)  # [3, 4] - Correct!

💡 Why Does This Happen?

Python uses an internal counter to track the current index in the loop. When you remove an element, the list shrinks, but Python doesn't update the counter. This causes elements to be skipped!

Solution: Iterate over a copy, modify the original.

🎯 Key Takeaways

Tuples

  • Immutable ordered sequences
  • Use parentheses: (1, 2, 3)
  • Perfect for fixed data and returning multiple values
  • Cannot be modified after creation

Lists

  • Mutable ordered sequences
  • Use square brackets: [1, 2, 3]
  • Can be modified: add, remove, change elements
  • More flexible but requires careful handling

Critical Concepts

  • Aliasing: Multiple variables can point to the same object
  • Side Effects: Modifying through one alias affects all aliases
  • Cloning: Use L[:] to create a copy
  • Mutation: Be careful when modifying lists during iteration

Best Practices

  • ✅ Iterate over elements directly when possible
  • ✅ Clone lists before modifying during iteration
  • ✅ Use sorted() if you don't want to mutate
  • ✅ Remember: most list methods return None
  • ✅ Use Python Tutor to visualize complex scenarios
