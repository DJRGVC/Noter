What Does a Computer Do?

At a fundamental level, computers do two things:

‚ö° Performs Calculations

Modern computers can perform a billion calculations per second. These calculations can be:

  • Built-in: Addition, subtraction, multiplication, etc.
  • User-defined: Calculations you create as a programmer
üíæ Remembers Results

Computers store results in memory. Modern computers have hundreds of gigabytes of storage available.

ü§ñ Critical Concept

Computers only know what you tell them.

Computers don't have minds. They don't make assumptions. They simply execute the instructions you provide, very quickly. If your program does something unexpected, it's because you told it to do that‚Äîeven if unintentionally.

Types of Knowledge

Declarative Knowledge

Declarative knowledge consists of statements of fact.

Example: "Someone will win a Google Cardboard before class ends."

This is a true statement, but it doesn't tell us how someone will win.

Imperative Knowledge

Imperative knowledge is a recipe or "how-to"‚Äîa sequence of steps to accomplish something.

Example: How to determine a raffle winner

  1. Students sign up for raffle
  2. Instructor opens their IDE
  3. Instructor chooses a random number between 1st and nth responder
  4. Instructor finds the number in the responders sheet. Winner!

Computers need imperative knowledge. They need step-by-step instructions to follow.

Algorithms: A Numerical Example

Let's explore the difference between declarative and imperative knowledge with a mathematical example: finding the square root of a number.

Declarative Approach

The square root of a number x is y such that y √ó y = x

This is true, but a computer doesn't know what to do with this statement.

Imperative Approach (Algorithm)

Here's a well-known algorithm to find the square root of 16:

g (guess) g √ó g x / g (g + x/g) / 2
3 9 5.33 4.17
4.17 17.36 3.837 4.0035
4.0035 16.0277 3.997 4.000002
üîÑ The Algorithm Steps
  1. Start with a guess g
  2. If g √ó g is close enough to x, stop and say g is the answer
  3. Otherwise, make a new guess by averaging g and x/g
  4. Using the new guess, repeat the process until close enough

What is a Recipe (Algorithm)?

A recipe or algorithm has three essential components:

1. Sequence of Steps

Simple, well-defined instructions

2. Flow of Control

Process that specifies when each step is executed (decisions, loops)

3. Stopping Condition

A means of determining when to stop

Sequence + Flow of Control + Stopping Condition = Algorithm

Computer Architecture

Fixed vs. Stored Program Computers

Fixed Program Computer

Example: Basic calculator

Can only perform predetermined operations (addition, subtraction, multiplication, division). To do something else, you need a different machine.

Stored Program Computer

Example: Modern computers

Can store and execute different sequences of instructions. Change the instructions, and the same machine does different tasks.

Basic Machine Architecture

Every computer has this fundamental architecture:

üñ•Ô∏è Core Components
  • Memory: Stores data and sequence of instructions
  • Control Unit: Contains program counter; manages instruction flow
  • Arithmetic Logic Unit (ALU): Performs primitive operations (add, subtract, compare, etc.)
  • Input/Output: Interfaces for data entry and results

How It Works

  1. Program counter starts at first instruction
  2. Control unit fetches instruction and sends to ALU
  3. ALU retrieves necessary data from memory
  4. ALU performs operation and may store result back to memory
  5. Program counter moves to next instruction
  6. Tests can change flow (skip instructions, loop back, etc.)
  7. Process continues until last instruction completes

Primitive Operations

The ALU can perform these basic operations:

  • Arithmetic: Addition, subtraction, multiplication, division
  • Logic: AND, OR, NOT operations
  • Tests: Equal to, less than, greater than comparisons
  • Data movement: Loading, storing, moving data
üí° Turing's Insight

Alan Turing proved that you can compute anything using just 6 primitive operations: move left, move right, read, write, scan, and do nothing.

Modern programming languages provide more convenient primitives, but the fundamental principle remains: anything computable in one language is computable in any other programming language.

  • Definitions are evaluated
  • Commands (statements) are executed by the Python interpreter

You can type commands directly in a shell (interactive mode) or store them in a file that is read and evaluated by the shell.

Objects and Types

Programs manipulate data objects. Every object has a type that defines what operations can be performed on it.

üé≠ Analogy

Think of types like species:

  • Human: Can walk, speak English, etc.
  • Wookie: Can walk, make Wookie sounds, etc.

Each type has specific capabilities and limitations.

Scalar Objects

Scalar objects cannot be subdivided. Python has four scalar types:

Type Description Examples
int Integers (whole numbers) 5, -3, 0, 1000
float Real numbers (decimals) 3.27, -0.5, 2.0
bool Boolean values True, False
NoneType Special type with one value None

Checking Types

Use the type() function to check an object's type:

Python

>>> type(3.0)

>>> type(True)

Type Conversions (Casting)

You can convert between types:

Python 
# Convert integer to float
float(3)  # Returns 3.0

# Convert float to integer (truncates, doesn't round)
int(3.9)  # Returns 3
‚ö†Ô∏è Important

Converting float to int truncates (removes decimal part), it does NOT round!

Operators and Expressions

Combine objects and operators to form expressions. Each expression has a value and a type.

üìù Expression Syntax

<object> <operator> <object>

Operators on Integers and Floats

Operator Operation Result Type
i + j Addition int if both int, else float
i - j Subtraction int if both int, else float
i * j Multiplication int if both int, else float
i / j Division Always float
i % j Remainder (modulo) int if both int, else float
i ** j Exponentiation (i to power j) int if both int, else float

Operator Precedence

Without parentheses, operations follow this order (highest to lowest):

  1. ** (exponentiation)
  2. * (multiplication)
  3. / (division)
  4. + and - (addition and subtraction, left to right)

Use parentheses to override default precedence:

Python 
3 + 5 * 2      # Returns 13 (multiplication first)
(3 + 5) * 2    # Returns 16 (parentheses first)

Variables and Assignments

Variables allow you to save values and reuse them throughout your program.

Assignment Syntax

The equal sign (=) is an assignment operator:

Python 
pi = 3.14159
pi_approx = 22 / 7
üîó Binding

Assignment binds a variable name to a value in memory. When you use that name later, Python retrieves the associated value.

Why Use Variables?

Reusability

Use names instead of repeating values throughout your code

Readability

Descriptive names make code easier to understand

Maintainability

Change value once, and it updates everywhere the variable is used

Example: Calculating Circle Area

Python 
pi = 3.14159
radius = 2.2
# Calculate area of circle
area = pi * (radius ** 2)

Programming vs. Math

‚ö†Ô∏è Critical Difference

In programming, you do NOT "solve for x"!

The equal sign is assignment, not equality:

  • Left side: Always a variable name
  • Right side: An expression that evaluates to a value
Python 
pi = 3.14159
radius = 2.2
# area of circle
area = pi * (radius ** 2)
radius = radius + 1  # This is valid! Rebinding radius

Rebinding Variables

You can change what value a variable points to:

Python 
pi = 3.14        # pi binds to 3.14
radius = 2.2     # radius binds to 2.2
area = pi * (radius ** 2)  # area binds to 15.1976
radius = radius + 1  # radius now binds to 3.2
üö® Important Concept

The value of area does NOT automatically change!

When you rebind radius, area keeps its old value (15.1976). The computer only does what you tell it to do. If you want to recalculate area, you must explicitly do so:

Python 
radius = radius + 1
area = pi * (radius ** 2)  # NOW area updates to new value

Memory Model

When you rebind a variable:

  • The old value may still exist in memory
  • But you've lost the handle to access it
  • Python's garbage collector will eventually clean up unused values
  • The variable now points to the new value
