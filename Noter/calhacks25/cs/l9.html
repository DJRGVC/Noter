Recap: Classes and Objects

Before we dive into new material, let's review the fundamental concepts of object-oriented programming from our previous lecture.

Abstract Data Types

Classes allow us to create abstract data types (ADTs) that bundle data and functionality together. We've seen examples like Coordinate and Fraction that abstract mathematical concepts into reusable Python objects.

Data Attributes

Variables that define what an object is. For example, a coordinate has x and y values.

Methods

Functions that define how to use an object. For example, calculating distance between coordinates.

Two Perspectives on Classes

When working with classes, we write code from two different viewpoints:

Implementing a Class

  • Define the class structure
  • Decide on data attributes (what IS the object)
  • Define methods (HOW TO use the object)
  • Use self to refer to instances generically

Using a Class

  • Create instances of the object type
  • Perform operations using methods
  • Access data through getters/setters
  • Don't worry about implementation details
🔑 Key Distinction

Class definition describes the type generically using self. An instance is one specific object with particular values. All instances share the same structure but have different data.

The Animal Class

Let's build a foundational example that we'll extend throughout this lecture. The Animal class represents a basic animal with an age and name.

Animal Class Definition Python 
class Animal(object):
    def __init__(self, age):
        self.age = age
        self.name = None
    
    def get_age(self):
        return self.age
    
    def get_name(self):
        return self.name
    
    def set_age(self, newage):
        self.age = newage
    
    def set_name(self, newname=""):
        self.name = newname
    
    def __str__(self):
        return "animal:" + str(self.name) + ":" + str(self.age)

Using the Animal Class

Creating and Using Animal Instances Python 
# Create a new animal
a = Animal(3)

# Access using dot notation (not recommended)
print(a.age)  # 3

# Better: use getter methods
print(a.get_age())  # 3

# Set the name
a.set_name("fluffy")
print(a)  # animal:fluffy:3

Getters and Setters

Getter and setter methods are fundamental to good object-oriented design. They provide controlled access to an object's data attributes.

Getters

Methods that return the value of data attributes. They provide read access to private data.

Setters

Methods that modify the value of data attributes. They provide controlled write access.

✅ Best Practice

Always use getters and setters to access data attributes from outside the class. This promotes information hiding and makes your code easier to maintain.

Information Hiding

One of the key principles of OOP is information hiding (also called encapsulation). Users of a class shouldn't need to know how it's implemented internally.

Why Information Hiding Matters

Consider what happens when a class implementation changes:

Original Implementation Python 
class Animal(object):
    def __init__(self, age):
        self.age = age  # Data attribute named 'age'
    
    def get_age(self):
        return self.age
Changed Implementation Python 
class Animal(object):
    def __init__(self, age):
        self.years = age  # Renamed to 'years'!
    
    def get_age(self):
        return self.years  # Getter updated

❌ Direct Access (Breaks)

a = Animal(3)
print(a.age)  # ERROR! No 'age' attribute

✅ Using Getter (Works)

a = Animal(3)
print(a.get_age())  # Still works!
⚠️ Python's Limitation

Python doesn't enforce information hiding like some other languages (e.g., Java's private keyword). Python allows you to:

  • Access data attributes directly: a.age
  • Modify data attributes: a.age = "infinite"
  • Create new attributes: a.size = "tiny"

Don't do any of these! It's bad style and breaks encapsulation.

Default Arguments

Default arguments allow methods to be called with fewer parameters, providing sensible defaults when arguments aren't supplied.

Method with Default Argument Python 
def set_name(self, newname=""):
    self.name = newname

Using Default

a = Animal(3)
a.set_name()  # Uses default ""
print(a.get_name())  # Prints empty string

Providing Argument

a = Animal(3)
a.set_name("fluffy")  # Overrides default
print(a.get_name())  # Prints "fluffy"

Hierarchies and Inheritance

One of the most powerful features of OOP is the ability to create hierarchies of classes. This allows us to model real-world relationships where some objects are specialized versions of others.

Animal

Parent and Child Classes

Parent Class (Superclass)

The base class that defines common attributes and behaviors. Example: Animal

Child Class (Subclass)

Inherits from parent and can add or override functionality. Example: Cat, Person

🔗 Inheritance Benefits
  • Inherit all data and methods from parent
  • Add new data attributes and methods
  • Override parent methods with specialized behavior
  • Reuse code without duplication

Example: Cat Class

Let's create our first subclass. A Cat is an Animal, so it inherits age and name, but adds its own behavior.

Cat Class Implementation Python 
class Cat(Animal):  # Inherits from Animal
    def speak(self):
        print("meow")
    
    def __str__(self):
        return "cat:" + str(self.name) + ":" + str(self.age)

What Cat Inherits

  • ✅ __init__ method (no need to redefine)
  • ✅ age and name data attributes
  • ✅ get_age(), get_name(), set_age(), set_name() methods

What Cat Adds/Overrides

  • ➕ speak() method (new functionality)
  • 🔄 __str__() method (overrides Animal's version)
Using the Cat Class Python 
c = Cat(5)
c.set_name("fluffy")
print(c)  # cat:fluffy:5
c.speak()  # meow
print(c.get_age())  # 5 (inherited method)

Method Resolution Order

When you call a method on an object, Python follows a specific search order to find the method:

🔍 Method Lookup Process
  1. Look for the method in the current class
  2. If not found, look in the parent class
  3. Continue up the hierarchy (grandparent, great-grandparent, etc.)
  4. Use the first method found with that name

This is why subclasses can override parent methods. If a method exists in both the child and parent, the child's version is found first and used.

Example: Person Class

The Person class demonstrates more advanced inheritance concepts, including calling the parent's __init__ and adding new data structures.

Person Class Implementation Python 
class Person(Animal):
    def __init__(self, name, age):
        Animal.__init__(self, age)  # Call parent's __init__
        self.set_name(name)
        self.friends = []  # New data attribute
    
    def get_friends(self):
        return self.friends
    
    def add_friend(self, fname):
        if fname not in self.friends:
            self.friends.append(fname)
    
    def speak(self):
        print("hello")
    
    def age_diff(self, other):
        diff = self.age - other.age
        print(abs(diff), "year difference")
    
    def __str__(self):
        return "person:" + str(self.name) + ":" + str(self.age)
💡 Code Reuse

Notice how Person.__init__ calls Animal.__init__. This is a common pattern: let the parent handle initialization it already knows how to do, then add your own specialized initialization.

Using the Person Class Python 
p1 = Person("jack", 30)
p2 = Person("jill", 25)

print(p1)  # person:jack:30
p1.speak()  # hello
p1.age_diff(p2)  # 5 year difference

p1.add_friend("jill")
print(p1.get_friends())  # ['jill']

Example: Student Class

The Student class shows multi-level inheritance: Student → Person → Animal. It also demonstrates using external libraries (the random module).

Student Class Implementation Python 
import random

class Student(Person):
    def __init__(self, name, age, major=None):
        Person.__init__(self, name, age)
        self.major = major
    
    def change_major(self, major):
        self.major = major
    
    def speak(self):
        r = random.random()  # Random float between 0 and 1
        if r < 0.25:
            print("i have homework")
        elif 0.25 <= r < 0.5:
            print("i need sleep")
        elif 0.5 <= r < 0.75:
            print("i should eat")
        else:
            print("i am watching tv")
    
    def __str__(self):
        return "student:" + str(self.name) + ":" + str(self.age) + ":" + str(self.major)

What Student Inherits

  • From Person: friends list, add_friend(), age_diff()
  • From Animal: age, name, getters and setters
Using the Student Class Python 
s1 = Student('alice', 20, "CS")
s2 = Student('beth', 18)  # major defaults to None

print(s1)  # student:alice:20:CS
print(s2)  # student:beth:18:None

s1.speak()  # Random output each time!
s2.speak()  # Random output each time!
student:alice:20:CS student:beth:18:None i need sleep i have homework

Class Variables

So far, we've worked with instance variables (like self.age). Now let's explore class variables, which are shared across all instances of a class.

Instance Variables

  • Unique to each instance
  • Defined with self.variable
  • Different values per object
  • Example: self.age

Class Variables

  • Shared by all instances
  • Defined in class body
  • Same value across all objects
  • Example: Rabbit.tag
⚠️ Important Distinction

When one instance modifies a class variable, all instances see the change. This is perfect for tracking information across all objects, like counting how many instances have been created.

Example: Rabbit Class

The Rabbit class demonstrates class variables, operator overloading, and working with object relationships.

Rabbit Class with Class Variable Python 
class Rabbit(Animal):
    tag = 1  # Class variable - shared across all instances!
    
    def __init__(self, age, parent1=None, parent2=None):
        Animal.__init__(self, age)
        self.parent1 = parent1
        self.parent2 = parent2
        self.rid = Rabbit.tag  # Access class variable
        Rabbit.tag += 1  # Increment for next rabbit
    
    def get_rid(self):
        return str(self.rid).zfill(3)  # Pad with zeros: 001, 002, etc.
    
    def get_parent1(self):
        return self.parent1
    
    def get_parent2(self):
        return self.parent2
    
    def __str__(self):
        return "rabbit:" + self.get_rid()

How Class Variables Work

Let's trace through what happens when we create rabbits:

Creating Rabbits Python 
r1 = Rabbit(3)  # Rabbit.tag = 1, r1.rid = 1, then tag becomes 2
r2 = Rabbit(4)  # Rabbit.tag = 2, r2.rid = 2, then tag becomes 3
r3 = Rabbit(5)  # Rabbit.tag = 3, r3.rid = 3, then tag becomes 4

print(r1)  # rabbit:001
print(r2)  # rabbit:002
print(r3)  # rabbit:003

Operator Overloading: Adding Rabbits

We can define what the + operator means for rabbits by implementing the __add__ method:

Rabbit Addition Python 
def __add__(self, other):
    # Return a new rabbit with age 0 and these two as parents
    return Rabbit(0, self, other)
Using Rabbit Addition Python 
r1 = Rabbit(3)
r2 = Rabbit(4)
r4 = r1 + r2  # Creates new rabbit with r1 and r2 as parents!

print(r4)  # rabbit:004
print(r4.get_parent1())  # rabbit:001
print(r4.get_parent2())  # rabbit:002

Comparing Rabbits

We can also define equality by implementing __eq__. Two rabbits are equal if they have the same parents:

Rabbit Equality Python 
def __eq__(self, other):
    # Compare parent IDs (not objects directly to avoid infinite recursion)
    parents_same = self.parent1.rid == other.parent1.rid \
                   and self.parent2.rid == other.parent2.rid
    parents_opposite = self.parent2.rid == other.parent1.rid \
                       and self.parent1.rid == other.parent2.rid
    return parents_same or parents_opposite
🚨 Important Note

We compare rid values (IDs) rather than rabbit objects directly. If we compared objects, it would call __eq__ recursively until reaching None, causing an AttributeError.

Testing Rabbit Equality Python 
r3 = Rabbit(5)
r4 = Rabbit(6)
r5 = r3 + r4  # Parents: r3, r4
r6 = r4 + r3  # Parents: r4, r3 (same parents, different order)

print(r5 == r6)  # True - same parents!
print(r4 == r6)  # False - different parents
