Introduction to Object Oriented Programming

Object-oriented programming (OOP) is a programming paradigm that organizes code around objects rather than functions and logic. If you haven't programmed before, this can be a challenging concept to grasp initially, but with practice and examples, it becomes a powerful tool in your programming toolkit.

💡 Core Concept

Everything in Python is an object. This fundamental principle means that all data types, functions, and even classes themselves are objects with specific properties and behaviors.

🎯 Type

Every object has a type that defines what kind of object it is (int, string, list, etc.)

📦 Data Representation

How Python internally stores and represents the object's data

⚙️ Interface

Methods and operations you can perform on the object

What Are Objects?

Objects are data abstractions that capture two essential components:

🚗 Real-World Analogy: Cars

Think of a car blueprint. Two cars can be created from the same blueprint but have different colors (blue vs. pink).

Internal Representation (Data Attributes):

  • Number of wheels
  • Number of doors
  • Length and height
  • Color

Interface (Methods):

  • Paint the car (change color)
  • Make a noise (honk)
  • Drive the car

Python Lists as Objects

Let's examine a familiar object type: the list [1, 2, 3, 4]

🔍 Internal Representation

Behind the scenes, Python represents a list as a linked list of cells. Each cell contains a value and a pointer to the next element's memory location.

1 → 2 → 3 → 4 →

How to Manipulate Lists:

  • L[i], L[i:j] - Indexing and slicing
  • len(), min(), max() - Built-in functions
  • L.append(), L.sort(), L.reverse() - Methods
⚠️ Important

The internal representation should be private. Directly manipulating internal representations can compromise the correct behavior of your objects.

Advantages of Object Oriented Programming

📦 Bundling

Package data and procedures together through well-defined interfaces

🔨 Divide & Conquer

Implement and test each class separately, reducing complexity

♻️ Reusability

Classes make code easy to reuse across different projects

🧬 Inheritance

Subclasses can extend or redefine superclass behaviors

Creating Your Own Classes

🎓 Two Perspectives

When working with classes, think about two distinct roles:

  • Implementing the class: Defining the class name and attributes
  • Using the class: Creating instances and performing operations

Defining a Class

Use the class keyword to define a new type:

Python 
class Coordinate(object):
    # Define attributes here
    pass
  • class - Keyword to define a new class
  • Coordinate - Name of your new type
  • object - Parent class (all classes inherit from object)
✨ Inheritance

Coordinate is a subclass of object, and object is a superclass of Coordinate.

Data Attributes

Data attributes are the pieces of information that make up an object. For a coordinate, these are the x and y values.

The __init__ Method

The __init__ method is a special method that Python calls automatically when you create a new instance of a class.

Python 
class Coordinate(object):
    def __init__(self, x, y):
        """ Sets the x and y values """
        self.x = x
        self.y = y
🔍 Understanding self
  • self - Refers to the instance of the object being created
  • x, y - Parameters passed when creating the object
  • self.x - Data attribute belonging to this instance

Important: self is always the first parameter in methods, but you don't pass it explicitly when calling methods.

Creating Instances

Python 
# Create coordinate objects
c = Coordinate(3, 4)
origin = Coordinate(0, 0)

# Access data attributes
print(c.x)        # Output: 3
print(origin.x)   # Output: 0
📝 Terminology

Data attributes of an instance are called instance variables.

Methods (Procedural Attributes)

Methods are functions that belong to a class and define how you can interact with objects of that class.

Defining a Method

Python 
class Coordinate(object):
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def distance(self, other):
        """ Returns the euclidean distance between two points """
        x_diff_sq = (self.x - other.x)**2
        y_diff_sq = (self.y - other.y)**2
        return (x_diff_sq + y_diff_sq)**0.5
🎯 Method Components
  • self - The instance calling the method
  • other - Another Coordinate object
  • self.x - Access data attributes using dot notation
  • Methods behave like functions: take parameters, perform operations, return values

Using Methods

Python 
c = Coordinate(3, 4)
zero = Coordinate(0, 0)

# Conventional way (recommended)
print(c.distance(zero))

# Equivalent explicit way
print(Coordinate.distance(c, zero))
💡 Pro Tip

The conventional way (c.distance(zero)) is preferred because it's more concise and readable. Python automatically passes c as the self parameter.

Special Methods

Special methods (also called "dunder" methods for double underscore) allow you to customize how your objects behave with Python's built-in operations.

The __str__ Method

By default, printing an object gives an uninformative representation:

Python 
c = Coordinate(3, 4)
print(c)
# Output: <__main__.Coordinate object at 0x7fa918510488>

Define __str__ to customize the print representation:

Python 
class Coordinate(object):
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __str__(self):
        """ Returns a string representation of self """
        return "<" + str(self.x) + "," + str(self.y) + ">"

c = Coordinate(3, 4)
print(c)  # Output: <3,4>
⚠️ Important

The __str__ method must return a string. Use str() to convert numbers to strings.

Other Special Operators

You can override many built-in operators to work with your custom classes:

Method Operator Description
__add__(self, other) self + other Addition
__sub__(self, other) self - other Subtraction
__eq__(self, other) self == other Equality
__lt__(self, other) self < other Less than
__len__(self) len(self) Length

Complete Example: Coordinate Class

Let's put everything together with a complete implementation of the Coordinate class:

Python 
class Coordinate(object):
    """ A coordinate made up of an x and y value """
    
    def __init__(self, x, y):
        """ Sets the x and y values """
        self.x = x
        self.y = y
    
    def __str__(self):
        """ Returns a string representation of self """
        return "<" + str(self.x) + "," + str(self.y) + ">"
    
    def distance(self, other):
        """ Returns the euclidean distance between two points """
        x_diff_sq = (self.x - other.x)**2
        y_diff_sq = (self.y - other.y)**2
        return (x_diff_sq + y_diff_sq)**0.5


# Using the Coordinate class
c = Coordinate(3, 4)
origin = Coordinate(0, 0)

print(c.x, origin.x)              # Output: 3 0
print(c.distance(origin))        # Output: 5.0
print(Coordinate.distance(c, origin))  # Output: 5.0
print(origin.distance(c))        # Output: 5.0
print(c)                         # Output: <3,4>

Type Checking

Python

# Output: <class '__main__.Coordinate'>

print(isinstance(c, Coordinate))
# Output: True

print(type(Coordinate))

Advanced Example: Fraction Class

Let's create a more sophisticated class that represents fractions and supports arithmetic operations:

Python 
class Fraction(object):
    """
    A number represented as a fraction
    """
    def __init__(self, num, denom):
        """ num and denom are integers """
        assert type(num) == int and type(denom) == int, "ints not used"
        self.num = num
        self.denom = denom
    
    def __str__(self):
        """ Returns a string representation of self """
        return str(self.num) + "/" + str(self.denom)
    
    def __add__(self, other):
        """ Returns a new fraction representing the addition """
        top = self.num * other.denom + self.denom * other.num
        bott = self.denom * other.denom
        return Fraction(top, bott)
    
    def __sub__(self, other):
        """ Returns a new fraction representing the subtraction """
        top = self.num * other.denom - self.denom * other.num
        bott = self.denom * other.denom
        return Fraction(top, bott)
    
    def __float__(self):
        """ Returns a float value of the fraction """
        return self.num / self.denom
    
    def inverse(self):
        """ Returns a new fraction representing 1/self """
        return Fraction(self.denom, self.num)

Using the Fraction Class

Python 
a = Fraction(1, 4)
b = Fraction(3, 4)

# Addition using the + operator
c = a + b  # c is a Fraction object
print(c)           # Output: 16/16

# Convert to float
print(float(c))   # Output: 1.0

# Explicit method call
print(Fraction.__float__(c))  # Output: 1.0

# Inverse and convert to float
print(float(b.inverse()))  # Output: 1.333...
🧮 How Addition Works

When you write a + b:

  1. Python looks for the __add__ method in the Fraction class
  2. Calculates: (1×4 + 4×3) / (4×4) = 16/16
  3. Returns a new Fraction object with numerator 16 and denominator 16
💡 Key Insight

Notice that __add__ returns a new Fraction object, not a number. This allows you to chain operations: (a + b).inverse()

Practical Example: IntSet Class

Let's implement a set of integers to demonstrate a more complex data structure:

Python 
class intSet(object):
    """
    An intSet is a set of integers
    The value is represented by a list of ints, self.vals
    Each int in the set occurs in self.vals exactly once
    """
    def __init__(self):
        """ Create an empty set of integers """
        self.vals = []

    def insert(self, e):
        """ Assumes e is an integer and inserts e into self """
        if not e in self.vals:
            self.vals.append(e)

    def member(self, e):
        """ Assumes e is an integer
        Returns True if e is in self, and False otherwise """
        return e in self.vals

    def remove(self, e):
        """ Assumes e is an integer and removes e from self
        Raises ValueError if e is not in self """
        try:
            self.vals.remove(e)
        except:
            raise ValueError(str(e) + ' not found')

    def __str__(self):
        """ Returns a string representation of self """
        self.vals.sort()
        return '{' + ','.join([str(e) for e in self.vals]) + '}'

Using the IntSet Class

Python 
s = intSet()
print(s)          # Output: {}

s.insert(3)
s.insert(4)
s.insert(3)      # Duplicate, won't be added
print(s)          # Output: {3,4}

print(s.member(3))  # Output: True
print(s.member(5))  # Output: False

s.insert(6)
print(s)          # Output: {3,4,6}

s.remove(3)
print(s)          # Output: {4,6}
🎯 Design Decisions
  • Internal representation: A list (self.vals)
  • Invariant: Each integer appears at most once
  • Methods: insert, member, remove, and string representation
  • Error handling: Raises ValueError when removing non-existent element

🎓 Key Takeaways

  • Everything in Python is an object with a type, internal representation, and methods
  • Classes define new types that bundle data (attributes) and behavior (methods)
  • The __init__ method initializes new instances of a class
  • The self parameter refers to the instance and is always the first parameter in methods
  • Special methods (like __str__, __add__) customize how objects interact with Python operators
  • Data abstraction separates implementation from interface, making code more maintainable
  • OOP enables code reuse through well-defined interfaces and inheritance
  • Build layers of abstraction by creating custom classes on top of Python's basic types
